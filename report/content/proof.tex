\section{Content}

\subsection{Useful definitions}

\begin{definition}[Global type]
An \textit{arrow} is a triple $(p,q,m)\in \mathbb{P}\times\mathbb{P}\times\mathbb{M}$
with $p\ne q$. We write \verb|Arr| to denote the finite set of arrows. 
A global type $G$ is a deterministic finite state automaton over the alphabet \verb|Arr|.
\end{definition}

\begin{definition}{\verb|rsc|-global type}
A global type $G$ is a \verb|rsc|-global type if it has no unmatched 
send events and there is a linearization $\rightsquigarrow$ where any 
matched send event is immediately followed by its respective receive event.
\end{definition}

\begin{definition}[Weakly-realizable]
Informally, realizable without deadlock-freedom.
More formally, in the \verb|rsc|-global type context, it's the 
weak closure $L^w$ of a set $L$ of \verb|rsc|-global type contains all the \verb|rsc|-global type $L$ weakly implies.
The set $L$ is weakly realizable iff $L=L^w$. The \verb|rsc|-global type-graph $G$ is said to be weakly
realizable if the set $L(G)$ of \verb|rsc|-global type is. Thus, a weakly realizable graph already contains
all the implied scenarios.
\end{definition}

\begin{definition}[Relaxed Post Correspondence Problem (RPCP) problem]
Given $\{(v_1, w_1), (v_2, w_2), ..., (v_r, w_r)\}$, determinate whether there are indices $i_1, ..., i_m$
such that $x_{i_1}...x_{i_n}=y_{i_1}...y_{i_m}$, where $x_{i_j}, y_{i_j} \in \{v_{i_j},w_{i_j}\}$, for some
index $i_l \  x_{i_l}\neq y_{i_l}$, and for all $j\leq m,y_{i_1}...y_{i_j}$ is a prefix of $x_{i_1}...x_{i_j}$.
The relaxed problem essentially says that must have the left side of the equivalence that grows more than the right side,
we must have also some tuple that are different, and that we can choose whatever side of tuple for the composition
of the string.
\end{definition}

RPCP is proven to be an undecidable problem by reduction from the classic
PCP problem by Alur, et al. \cite{alur2005realizability}.

\subsection{Proof}

\begin{theorem}
Given a \verb|rsc|-global type $G$, checking if $G$ is (weakly)-realizable is undecidable.
\end{theorem}

\begin{proof}
The proof proceeds via a reduction from the RPCP problem. 

Given a finite set $L$ of \verb|rsc|-global type, 
let $L^*$ denote the \verb|rsc|-global type graph that consists of the \textit{complete}
graph with $|L|$ vertices, one per \verb|rsc|-global type in the set $L$, dummy initial
and terminal vertices $v_I$, $v_T$ with empty transition, and edges from
$v_I$ to all vertices of $L$ and from those to $v_T$. Thus, a \verb|rsc|-global type of
this graph is simply a concatenation of \verb|rsc|-global type from the set $L$.

Given an instance $\Delta = \{(v_1, w_1), \ldots, (v_m, w_m)\}$ of
RPCP, we build a set $L$ of \verb|rsc|-global type over 4 processes as follows.
For each pair $(v_i, w_i)$ we build a global type $G^n_i$,
which is depicted in Fig.~\ref{fig:gtype}. The double arrow denote that 
an ACK message consisted of the same message is sent.
Thus, in $G^n_i$ where $n\in\{0,1\}$, process $q$ sends $(i, n)$ to process $q$,
then sends the index $i$ to process 4, and then process 4 sends $(i, 0)$ to process 3. 
After that, process 2 sends the sequence of characters of $v_i$ to process 3 
(note we assume $c$ is the length of $v_i$ and $d$ the length of $w_i$
in the figure), $G^1_i$ is similar. Observe that the communication graph of each of these
global types are strongly connected and involves all the processes, and
hence, the global type graph $L^*$ is bounded.

We need to prove
\begin{center}
    $\Delta \in \text{RPCP}$ if and only if the global type $L^*$ is not realizable.
\end{center}

First, we prove that if we have a solution of RPCP we can create $G_i^n$.
All the proof follow the one of Alur. The only additional claim that we need to 
prove is that the created global type is a \verb|rsc|-global type.

\textbf{Claim 1:} $G_i^n$ is a \verb|rsc|-global type.
\begin{proof}
We just need to prove that all the possible linearization can be considered synchronous.
Thanks to the ACK message, we have almost always the corresponding receive after the send.
The only receive that can ``move'' is the ACK message from \arrmess{p}{s}{i}.

\end{proof}
